🚀 Automatisiertes Mac‑Setup (Terminal‑Script)

🔧 Was das Script macht:• prüft Homebrew

• installiert Git (falls nötig)

• setzt Git‑Konfiguration

• erzeugt SSH‑Key

• aktiviert SSH‑Agent

• kopiert Public Key in die Zwischenablage

• aktiviert VS‑Code‑CLI

• legt Dev‑Ordner an

• testet GitHub‑Verbindung

📦 1. Script kopieren & im Terminal ausführen

#!/bin/bash

echo "🚀 Starte automatisiertes Mac-Setup..."

# -------------------------------

# 1. Homebrew prüfen

# -------------------------------

if ! command -v brew &> /dev/null

then

    echo "🍺 Homebrew nicht gefunden. Installation wird gestartet..."

    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

else

    echo "🍺 Homebrew ist bereits installiert."

fi

# -------------------------------

# 2. Git installieren (falls nötig)

# -------------------------------

if ! command -v git &> /dev/null

then

    echo "🔧 Git wird installiert..."

    brew install git

else

    echo "🔧 Git ist bereits installiert."

fi

# -------------------------------

# 3. Git konfigurieren

# -------------------------------

echo "🧩 Git konfigurieren..."

git config --global user.name "Volker"

git config --global user.email "DEINE_EMAIL@domain.de"

git config --global init.defaultBranch main

# -------------------------------

# 4. SSH-Key erzeugen

# -------------------------------

SSH_KEY=~/.ssh/id_ed25519

if [ ! -f "$SSH_KEY" ]; then

    echo "🔑 Erzeuge neuen SSH-Key..."

    ssh-keygen -t ed25519 -C "DEINE_EMAIL@domain.de" -f $SSH_KEY -N ""

else

    echo "🔑 SSH-Key existiert bereits."

fi

# -------------------------------

# 5. SSH-Agent starten & Key hinzufügen

# -------------------------------

echo "🔐 SSH-Agent starten..."

eval "$(ssh-agent -s)"

ssh-add $SSH_KEY

# -------------------------------

# 6. Public Key in Zwischenablage kopieren

# -------------------------------

echo "📋 Public Key wurde in die Zwischenablage kopiert!"

pbcopy < ${SSH_KEY}.pub

# -------------------------------

# 7. VS Code CLI aktivieren

# -------------------------------

if command -v code &> /dev/null

then

    echo "💡 VS Code CLI ist bereits aktiv."

else

    echo "💡 VS Code CLI wird aktiviert..."

    cat <<EOF

Bitte öffne VS Code und führe aus:

Cmd + Shift + P → "Shell Command: Install 'code' command in PATH"

EOF

fi

# -------------------------------

# 8. Dev-Ordner anlegen

# -------------------------------

echo "📁 Lege ~/dev an..."

mkdir -p ~/dev

# -------------------------------

# 9. GitHub-Verbindung testen

# -------------------------------

echo "🧪 Teste SSH-Verbindung zu GitHub..."

ssh -T git@github.com

echo "🎉 Setup abgeschlossen!"

#!/bin/bash

echo "🚀 Starte erweitertes Mac-Setup..."

# -------------------------------

# 1. Homebrew installieren

# -------------------------------

if ! command -v brew &> /dev/null

then

    echo "🍺 Homebrew wird installiert..."

    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

else

    echo "🍺 Homebrew ist bereits installiert."

fi

echo "🔄 Homebrew aktualisieren..."

brew update

# -------------------------------

# 2. Wichtige Pakete installieren

# -------------------------------

echo "📦 Installiere Standard-Entwicklungspakete..."

brew install git

brew install node

brew install python

brew install docker

brew install wget

brew install gh   # GitHub CLI

# -------------------------------

# 3. Git konfigurieren

# -------------------------------

echo "🧩 Git konfigurieren..."

git config --global user.name "Volker"

git config --global user.email "DEINE_EMAIL@domain.de"

git config --global init.defaultBranch main

# -------------------------------

# 4. SSH-Key erzeugen

# -------------------------------

SSH_KEY=~/.ssh/id_ed25519

if [ ! -f "$SSH_KEY" ]; then

    echo "🔑 Erzeuge neuen SSH-Key..."

    ssh-keygen -t ed25519 -C "DEINE_EMAIL@domain.de" -f $SSH_KEY -N ""

else

    echo "🔑 SSH-Key existiert bereits."

fi

echo "🔐 SSH-Agent starten..."

eval "$(ssh-agent -s)"

ssh-add $SSH_KEY

echo "📋 Public Key wurde in die Zwischenablage kopiert!"

pbcopy < ${SSH_KEY}.pub

# -------------------------------

# 5. Ordnerstruktur anlegen

# -------------------------------

echo "📁 Lege Entwicklungsordner an..."

mkdir -p ~/dev/projects

mkdir -p ~/dev/tools

mkdir -p ~/dev/tmp

# -------------------------------

# 6. VS Code installieren (falls nicht vorhanden)

# -------------------------------

if [ ! -d "/Applications/Visual Studio Code.app" ]; then

    echo "💻 VS Code wird installiert..."

    wget -O ~/Downloads/vscode.zip "https://update.code.visualstudio.com/latest/darwin/universal/stable"

    unzip ~/Downloads/vscode.zip -d /Applications

else

    echo "💻 VS Code ist bereits installiert."

fi

# -------------------------------

# 7. VS Code CLI aktivieren

# -------------------------------

if ! command -v code &> /dev/null

then

    echo "⚙️ Bitte VS Code öffnen und CLI aktivieren:"

    echo "Cmd + Shift + P → 'Shell Command: Install code command in PATH'"

else

    echo "⚙️ VS Code CLI ist aktiv."

fi

# -------------------------------

# 8. VS Code Extensions installieren

# -------------------------------

echo "🧩 Installiere VS Code Erweiterungen..."

code --install-extension GitHub.copilot

code --install-extension GitHub.vscode-pull-request-github

code --install-extension eamodio.gitlens

code --install-extension esbenp.prettier-vscode

code --install-extension ms-python.python

code --install-extension ms-azuretools.vscode-docker

code --install-extension antropic.claude-code

# -------------------------------

# 9. ZSH optimieren (Aliases + Quality-of-Life)

# -------------------------------

echo "⚡ Optimiere ZSH..."

cat << 'EOF' >> ~/.zshrc

# --- Custom Aliases ---

alias gs="git status"

alias ga="git add ."

alias gc="git commit -m"

alias gp="git push"

alias gl="git pull"

alias v="code ."

alias dev="cd ~/dev/projects"

# --- Better LS ---

alias ll="ls -lah"

# --- Node & Python Shortcuts ---

alias py="python3"

alias npmi="npm install"

EOF

# -------------------------------

# 10. GitHub CLI vorbereiten

# -------------------------------

echo "🔗 GitHub CLI vorbereiten..."

echo "Bitte führe danach aus: gh auth login"

# -------------------------------

# 11. Docker starten

# -------------------------------

echo "🐳 Docker Desktop muss manuell installiert werden:"

echo "https://www.docker.com/products/docker-desktop/"

# -------------------------------

# 12. Abschluss

# -------------------------------

echo "🎉 Erweitertes Setup abgeschlossen!"

echo "👉 SSH-Key ist in der Zwischenablage – füge ihn jetzt bei GitHub ein."

**Claude Setup und Start**

 

davon ausgehend, dass bei Dir Claude bereits in VSCode läuft, habe ich Dir hier eine kleine Setup-Installation erstellt, welche die aus meiner Sicht sinnvollen Tools und Plugins einrichtet. Damit läuft Claude dann sozusagen im professionellen Modus, was später sehr hilfreich ist. Vorteil - Du musst den ganzen Mist nicht selber installieren.

Das Ganze kannst Du alleine machen - in Vorbereitung auf unsere Session oder deine eigene ersten Versuche...

 

Vorgehen:

- Zip entpacken (enthält zwei dateien - die md-Datei mit allen Anweisungen für Claude und ne Readme - die kannst du dir mal durchlesen. Da steht drin was es macht.)
- Dann einen neuen Ordner für ein neues (leeres) Claude projekt anlegen. Irgendwo - wo du deine ganzen Claude-Projekte speichern willst (Beispiel von mir: C:\Users\ctipp\Documents\Claude Code Projekte\NEUES PROJEKT VERZEICHNIS)
- VSCode öffnen und oben übers Menü das neue Verzeichnis öffnen (Bei mir in Englisch: File --> open Folder --> dann auswählen). Damit befindet sich VSCode in dem neuen, leeren Projekt Ordner und Claude kann dort arbeiten
- In den Ordner die beiden Dateien reinkopieren
- Dann neues Terminal öffnen (via Menü oder Tastenkürzel)
- dort "Claude" eingeben, damit claude startet (der wird  dich immer erst fragen, ob du dem Verzeichnis vertraust. Was Du natürlich tust.) und dann einfach schreiben "Folge InitialSetup.md"

 

Damit startet Claude mit einem interaktiven Setup - er wird dich dann ein paar Dinge fragen. Aber alles soweit selbst machen. Git (lokale Versionskontrolle) und Github (Online Repository (damit legt er deinen Code auf Github und du hast dort ein Backup) - immer auf private stellen lassen) solltest du auch einrichten lassen.
