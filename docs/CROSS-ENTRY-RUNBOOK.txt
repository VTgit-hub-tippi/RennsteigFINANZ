CROSS-ENTRY-RUNBOOK (APP- UND PROZEDUR-UNABHAENGIG)
Stand: 2026-07-09

Ziel:
Jeder Einstieg soll von jedem Tool aus moeglich sein (VS Code, Claude.ai, Browser, andere Assistenten), ohne Kontextverlust.

PRINZIP:
Nicht der Chat ist Wahrheit, sondern die Projektdateien + Git.

BETRIEBSVERHALTEN DER ASSISTENTEN (verbindlich):
1) Erst Systempruefung, dann Inhalt.
2) Bei fehlenden Quellen: OFFEN markieren, keine Ersatzannahmen.
3) Keine Websuche als Ersatz fuer lokale Projektquellen.
4) Sprache in klarem, gut verstaendlichem Deutsch; Fachbegriffe kurz erklaeren.
5) Anrede immer in der Sie-Form.

MINIMALPAKET (immer gleich):
1) CLAUDE.md
2) STATUS.md
3) NEXT-ACTION.md
4) docs/context-snapshot.md
5) git status -sb

AUTO-PROOF (ohne manuelle Checkliste):
- Ein Befehl: `bash tools/handoff-proof.sh`
- Ergebnis: PASS/FAIL fuer Vollstaendigkeit, Referenzen, Git-Grundzustand.

AUSSTIEGSPROZESS (vor Toolwechsel):
1) STATUS.md kurz fortschreiben (was erledigt, was offen).
2) NEXT-ACTION.md auf genau eine naechste Aktion setzen.
3) docs/context-snapshot.md aktualisieren.
4) git status -sb pruefen und als Zustand merken.

EINSTIEGSPROZESS (in neuem Tool):
1) Minimalpaket einlesen.
2) Rekonstruktion in 5-8 Punkten erzeugen.
3) Genau eine P0-Aktion festlegen.
4) Erst dann weiterarbeiten.

NOTFALLREGEL:
Wenn Informationen fehlen: OFFEN markieren, keine Halluzination, keine stillen Annahmen.
